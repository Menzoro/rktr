What this tests: whether your WebView honours timestamped send().

1. Set Minilogue Clock Source = Auto (or External) in global settings.
2. Plug in via USB, hit Connect MIDI, confirm the port auto-selects.
3. Set a BPM, hit Start. The Minilogue's arp/sequencer/delay should lock to it.
4. Watch measured tempo — if it sits steady on your target BPM, timestamps work. If it jumps around, the WebView is ignoring timestamps (the risk we flagged).

Uses performance.now() as the clock base — same time domain as MIDI timestamps, so this isolates the timestamp question from the audio-clock question.
